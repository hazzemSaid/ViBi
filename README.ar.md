## موارد التطوير

هذا التطبيق مبني باستخدام Flutter ومكتوب بلغة Dart.

يعتمد على خدمات Supabase (المصادقة، PostgreSQL، التخزين، والميزات اللحظية)، بالإضافة إلى واجهات GraphQL لوظائف الشبكة الاجتماعية والخلاصة.


### الميزات الأساسية

- مصادقة المستخدمين (البريد الإلكتروني/كلمة المرور + Google OAuth)
- ملفات شخصية مع صور رمزية وروابط اجتماعية
- القصص (محتوى يختفي خلال 24 ساعة)
- صندوق أسئلة وأجوبة (أسئلة مجهولة)
- نظام الخلاصة مع المتابعة
- نظام المتابعة وإلغاء المتابعة
- البحث (عن المستخدمين والمحتوى)
- الإشعارات مع دعم الإشعارات الفورية

### قيد التطوير

- الرسائل المباشرة
- إدارة المحتوى
- خوارزمية المحتوى الرائج
- العمل دون اتصال

### المعمارية

يتبع ViBi مبادئ Clean Architecture:

- طبقة العرض: واجهة المستخدم وإدارة الحالة (Bloc/cubit)
- طبقة المجال: منطق العمل والكيانات
- طبقة البيانات: مصادر البيانات (Supabase) والمستودعات

### التقنيات المستخدمة

- الواجهة الأمامية: Flutter + Dart
- إدارة الحالة: Bloc/cubit
- الخلفية: Supabase (PostgreSQL)
- المصادقة: Supabase Auth
- التوجيه: GoRouter
- واجهات API: GraphQL

## المساهمات

### ملاحظة

نحن نقبل المساهمات، لكننا نعطي الأولوية للمشكلات وطلبات الدمج عالية الجودة. الالتزام بالإرشادات التالية يساعدنا على مراجعة مساهمتك بشكل أسرع.

### القواعد

- قد لا نرد على المشكلة أو طلب الدمج الخاص بك.
- قد نغلق المشكلة أو طلب الدمج دون تفاصيل كثيرة.
- قد نغلق النقاشات أو المساهمات إذا تعرضنا لضغط كبير أو إغراق.
- لن نقدم دعماً لمشكلات البناء والتشغيل.

### الإرشادات

- تحقق من [المشكلات الموجودة](https://github.com/yourusername/vibi/issues) قبل فتح مشكلة جديدة.
- افتح مشكلة واترك وقتاً للنقاش قبل إرسال طلب دمج.
- تجنب طلبات الدمج مثل:
  - تغيير المصطلحات العامة على مستوى المشروع بالكامل.
  - إعادة هيكلة شاملة للكود، مثل استبدال Bloc/cubit بـ Provider.
  - إضافة ميزات جديدة بالكامل بدون نقاش مسبق.

نحن نخدم مجتمعاً واسعاً، وأولوياتنا تتغير باستمرار. إذا قدمت طلبات دمج مكتوبة جيداً وتحل مشكلة فعلية باختصار، فهذه مساهمة ممتازة. غير ذلك، وبكل تقديرنا لجهدك، قد لا نملك الوقت لاستيعاب كل المقترحات. لهذا السبب التفريع (Forking) خيار ممتاز.

### نطاق المساهمة

المساهمات العامة تركز حالياً على الواجهة الأمامية:

- تحسينات UI وUX
- إصلاحات وميزات Flutter في كود العميل
- الاختبارات وتحديثات التوثيق

خارج نطاق طلبات الدمج العامة (إلا إذا طلب فريق الصيانة ذلك):

- تغييرات مخطط قاعدة البيانات وملفات الهجرة
- تعديلات البنية الخلفية وبيانات اعتماد الإنتاج
- سياسات الأمان وإعدادات النشر للإنتاج

### معايير الكود

- اتبع [دليل أسلوب Dart](https://dart.dev/guides/language/effective-dart/style)
- استخدم رسائل commits القياسية: `feat:`, `fix:`, `docs:`, `test:`, `refactor:`
- Null safety إلزامي
- استخدم Bloc/cubit لإدارة الحالة
- أضف اختبارات للميزات الجديدة
- حدّث التوثيق عند تغيير واجهات API

## إرشادات التفريع

لديك كامل الحرية في تفريع هذا التطبيق، لكن من المهم جداً أن يكون واضحاً للمستخدمين أنهم يستخدمون نسخة متفرعة.

يرجى التأكد من:

- تغيير الهوية والعلامة التجارية في المستودع والواجهة لتكون مختلفة بوضوح عن ViBi.
- تغيير روابط الدعم (التعليقات، البريد، الشروط، وغيرها) إلى أنظمتك الخاصة.
- استبدال أدوات التحليلات وتتبع الأخطاء بأدواتك الخاصة لتجنب التداخل.
- توضيح أن النسخة هي تفريع من ViBi.

## الإفصاح الأمني

إذا اكتشفت ثغرة أمنية، يرجى إرسال بريد إلى **security@vibi-app.dev**. سيتم نسخ فريق العمل تلقائياً وسنرد في أقرب وقت.

## هل أنت مطور مهتم بالبناء باستخدام Supabase وFlutter؟

ViBi تطبيق أسئلة وأجوبة اجتماعي مفتوح المصدر مبني على Supabase وFlutter. يمنحك Supabase قاعدة بيانات PostgreSQL قابلة للتوسع، ونظام مصادقة، وميزات لحظية، وأكثر، لتبني تطبيقات قوية بدون الارتباط بمنصات مغلقة.
